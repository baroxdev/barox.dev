This is the first entry in the journal, and fittingly it's about the site
itself. I'm building barox.dev on TanStack Start, mostly to practice writing
and to have somewhere to put lessons learned as I go.

## Why a content pipeline first

Before any page renders a single post, there needs to be a trustworthy way to
turn an MDX file into something the rest of the site can rely on.

<Sidenote>
  That's the whole point of the content pipeline: parse once, validate hard, and
  let every downstream page trust the shape of a Post.
</Sidenote>
Frontmatter gets validated with a schema, so a typo'd date or a missing tag
fails immediately instead of turning into a broken page three routes later.

Here's the rough shape of a parsed post:

```ts title="types.ts"
interface Post {
  slug: string
  title: string
  date: Date
  tags: string[]
  published: boolean
  excerpt: string
}
```
